# pnCases 1.4.3

Версия 1.4.3 делает pnCases удобнее для настройки и приятнее для игроков. Главный упор сделан на меню настройки кейса, несколько блоков одного кейса, новую анимацию и более понятное поведение в GUI.

## Что нового

- Добавлено меню настройки кейса через `/pncases machine`.
- В machine-меню появились отдельные разделы:
  - анимация кейса;
  - голограмма;
  - внешний вид меню;
  - покупка ключа за опыт;
  - разметка слотов в GUI.
- Теперь многие настройки можно менять прямо в игре, без постоянного ручного копания в config.yml.
- Добавлена настройка разметки меню кейса: можно менять декор, кнопку открытия, историю, кнопку анимации и другие элементы.
- Добавлена настройка покупки ключа за опыт: можно включать, отключать и менять цену.
- Добавлена новая анимация открытия: `Круг фортуны`.
- Один и тот же кейс теперь можно поставить на несколько блоков обычной командой `/pncases setcase <кейс>`.
- Примеры кейсов вынесены в отдельные файлы `plugins/pnCases/cases/`, чтобы основной `config.yml` был короче и понятнее.
- Добавлены готовые примеры кейсов для разных типов наград: деньги, PlayerPoints, предметы и LuckPerms.

## Улучшено

- `/pncases setcase <кейс>` теперь добавляет новый блок к кейсу, а не заменяет старый.
- `/pncases delcase <кейс>` оставлен как одна понятная команда для удаления блоков кейса.
- Лишние варианты команд удаления убраны, чтобы не было путаницы.
- Содержимое кейса теперь удобнее открывать через основную кнопку кейса.
- Если покупка ключа за опыт отключена, в подсказке больше не пишется, что покупка отключена. Вместо этого игрок видит понятное действие для просмотра содержимого.
- Если у игрока нет ключей, в меню теперь нормально написано `Недостаточно ключей`.
- Старые настройки кейсов читаются аккуратнее, чтобы обновление было проще.
- Новый список блоков кейса поддерживает старый формат `block:` и новый формат `blocks:`.
- README и update-manifest обновлены под версию 1.4.3.

## Исправлено

- Исправлены голограммы для нескольких блоков одного кейса.
- У каждого блока кейса теперь своя голограмма, они больше не перетирают друг друга.
- При открытии кейса скрывается только голограмма того блока, который реально открыл игрок.
- После окончания открытия голограмма возвращается обратно на нужный блок.
