# pnCases 1.5.0

Версия 1.5.0 — крупное обновление Machine, витрин и анимаций. Главная цель релиза — сделать настройку кейсов понятнее, интерфейсы аккуратнее, а объекты в мире стабильными при изменениях конфигурации, unload, reload и выключении плагина.

## Новое

### Machine и настройка кейсов

- Полностью переработано главное меню `/pncases machine`: убран лишний предпросмотр, упрощена навигация и пересобрана витрина кейса.
- Название кейса в Machine берётся из ключа, а заголовки поддерживают RGB-оформление.
- Базовые типы кейсов теперь выбираются в интерфейсе: деньги, предметы, PlayerPoints и LuckPerms.
- `/pncases createcase <id>` создаёт кейс типа `money` и автоматически добавляет для него ключ в `config.yml`.
- `/pncases setcase <id>` больше не создаёт новый кейс по опечатке: сначала кейс необходимо создать отдельной командой.
- После создания или выбора кейса команда подсказывает дальнейший путь: `/pncases machine` для настройки.
- Оформление Machine перенесено из `gui.yml` в код. В конфиге больше нет дублирующих и конфликтующих настроек `machine`.

### Интерфейсы

- Переработаны выбор кейса, выбор игрока и выбор анимации: компактный lore, понятные действия и аккуратное расположение кнопок.
- Настройка витрины очищена от лишних технических параметров и дублей иконок.
- `air` в `gui.yml` теперь можно использовать как пустой слот.
- Предметам GUI скрываются атрибуты: урон, скорость атаки и другие служебные подсказки не должны мешать дизайну.
- Добавлена плавная анимация открытия интерфейсов: декор и кнопки появляются постепенно.
- Динамические обновления GUI работают без полного переоткрытия окна и лишнего мигания; при наличии ProtocolLib обновляются только нужные слоты.
- При выключении pnCases все открытые меню корректно закрываются.

### Новые анимации

- Добавлена **Mob Hunt** (`MOB_HUNT`, «Охота на стражей») — режим открытия с мобами вокруг кейса.
- В Machine новые режимы отмечены тегом `NEW`; в обычном меню выбора игрока технические теги не выводятся.
- При выборе режима с мобами администратор получает отдельное служебное предупреждение; обычным игрокам оно не показывается.
- Добавлена «Водяная капсула»: кейс временно превращается в стеклянную капсулу с сердцем моря, вокруг движутся водные эффекты и призмариновые частицы, затем награда плавно появляется из воды.
- Размер головы сердца моря увеличен, а сама анимация дополнена движением и эффектами, чтобы сцена не выглядела статичной.

### Хранилище и форматирование

- Поддерживаются SQLite, MySQL и MongoDB.
- Через pnLibrary доступны MiniMessage, обычные цвета `&a` и `§a`, а также HEX-формат `&#RRGGBB`.

## Исправления и стабильность

- Исправлена проблема, при которой изменение Machine или анимации одного кейса заставляло hologram других кейсов пропадать и появляться заново.
- Обновления теперь применяются точечно к нужному кейсу: чужие витрины и hologram не пересоздаются.
- Смена базового типа в Machine больше не перезаписывает весь кейс. Сохраняются его ID, ключ, название, GUI, hologram, витрина, анимация и привязанные блоки; меняются только тип и награды.
- Один блок теперь может принадлежать только одному кейсу. `/pncases setcase` не затирает чужую привязку и сообщает, что блок уже занят.
- Загрузка кейсов стала безопасной: повреждённый YAML, дубликат ID или конфликт блоков отменяют reload, но не удаляют уже работающие кейсы из памяти.
- ID отдельного кейса канонически берётся из имени `cases/<id>.yml`; несовпадающее поле `id` больше не может незаметно подменить другой кейс.
- Запись `config.yml` и `cases/*.yml` выполняется через временный файл с повторным чтением, резервной копией `.bak` и атомарной заменой. При ошибке команда больше не показывает ложный успех.
- Миграция из секции `config.yml:cases` в `cases/*.yml` больше не теряет привязки: каждый кейс экспортируется и перечитывается до удаления старой секции; предыдущие standalone-файлы сохраняются в `.bak`.
- В маленьких GUI исправлены коллизии слотов: `open`, `animation`, `preview`, `history` и `decor` больше не перезаписывают друг друга, а кнопка открытия сохраняет приоритет.
- Экраны Machine после изменения обновляют только изменившиеся слоты и не запускают повторно анимацию появления.
- `Pillager Raid`, «Круг фортуны» и «Водяная капсула» хранят полный `BlockState` и восстанавливают исходный блок при завершении, отмене, unload мира и выключении плагина.
- Усилена очистка объектов витрины: при unload, load, reload и выключении плагина не должны оставаться предметы или display-сущности над кейсом.
- Устранены дубли названий анимаций и лишние элементы в меню настроек.

## Совместимость

- Paper и Purpur 1.16.5–1.21.11.
- Java 17 или новее.
- На современных версиях используются Display-сущности и расширенные эффекты. Для старых версий сохраняется безопасный запасной режим анимаций.
- LuckPerms, Vault, PlayerPoints, FancyHolograms, DecentHolograms и ProtocolLib подключаются только для соответствующих возможностей.

## Установка

1. Скачайте `pnCases-1.5.0.jar`.
2. Замените старый JAR в папке `plugins/`.
3. Полностью перезапустите сервер.
4. Существующие кейсы и данные сохраняются. Для новых возможностей интерфейса используйте актуальные `config.yml` и `gui.yml` из релиза.
