# pnCases 1.4.4

Версия 1.4.4 - небольшой hotfix после 1.4.3. Основной упор: убрать баг с возвращающимися часами истории и дать нормально настраивать нижние подсказки в GUI кейса.

## Что исправлено

- Исправлено удаление истории в разметке GUI.
- Если в machine-меню убрать все слоты истории, часы больше не возвращаются после обновления меню, `/pncases reload` или перезапуска сервера.
- Пустой список `history.slots: []` теперь считается нормальной настройкой, а не причиной вернуть стандартные слоты.

## Что улучшено

- Нижний блок подсказок на кнопке кейса теперь настраивается через `messages.yml`.
- Можно менять строки про ключи, ЛКМ, ПКМ, СКМ и порядок этих строк.
- Можно полностью убрать этот блок, если поставить `extra-lore: []`.
- Добавлены плейсхолдеры для подсказок GUI:
  - `{have}` - сколько ключей есть у игрока;
  - `{need}` - сколько ключей нужно;
  - `{levels}` - цена покупки ключа за опыт;
  - `{key}` - ID ключа;
  - `{key_name}` / `{key-name}` - красивое имя ключа;
  - `{material}` - материал кнопки кейса;
  - `{keys-balance}` - готовая строка баланса ключей;
  - `{left-click}` - действие ЛКМ;
  - `{right-click}` - действие ПКМ;
  - `{middle-click}` - действие СКМ.
- Lore временного barrier-предмета при попытке открыть кейс без ключей тоже вынесен в `messages.yml`.
- Старые `messages.yml` автоматически получают новые отсутствующие настройки, но уже измененные строки не перезаписываются.

## Где настраивать

```yaml
gui:
  open:
    no-keys-lore:
      - "&cНедостаточно ключей"
      - "{keys-balance}"
      - "&7Получите ключ и попробуйте снова."
  case-button:
    extra-lore:
      - ""
      - "{keys-balance}"
      - ""
      - "{left-click}"
      - "{right-click}"
      - "{middle-click}"
```
