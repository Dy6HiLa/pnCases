Возможности

Полная система кейсов без лишних ограничений

pnCases закрывает весь цикл: от создания кейса и настройки наград до эффектного открытия, хранения истории и автоматической проверки обновлений.

Гибкие награды

Предметы, команды, Vault, PlayerPoints и LuckPerms с шансами, редкостями и отдельным визуальным представлением.

Премиальные анимации

Наковальня, динамит, портал, отравление и астральный разлом с автоматическим режимом совместимости.

Интерфейсы и preview

Удобный GUI, предварительный просмотр наград, выбор анимации и понятная навигация для игроков.

История открытий

Результаты открытий и пользовательские настройки сохраняются в надёжной локальной базе SQLite.

Витрины кейсов

Предмет над свободным кейсом, настраиваемые эффекты, темы частиц и ограничение дистанции отображения.

Голограммы

Автоматический выбор FancyHolograms или DecentHolograms без жёсткой привязки к одному провайдеру.

Автомиграция данных

Старые YAML-файлы переносятся в SQLite автоматически, без ручной конвертации и потери данных.

Проверка обновлений

GitHub Release, git tags, update-manifest и plugin.yml проверяются автоматически при запуске.

Установка

Запуск за четыре шага

Для базового запуска нужен только совместимый Paper или Purpur. Остальные зависимости подключаются по необходимости.

Скачайте плагин

Загрузите актуальный файл pnCases-1.4.6.jar.

Переместите JAR

Поместите файл в серверную директорию plugins/.

Перезапустите сервер

Выполните полный перезапуск, чтобы плагин создал необходимые файлы и базу данных.

Настройте конфигурацию

Измените config.yml и messages.yml под свой сервер.

Управление

Команды администратора

Все административные действия защищены правом pncases.admin.

Команда Описание
/pncases Показывает команды, версию и статус обновления
/pncases reload Перезагружает config.yml и messages.yml
/pncases setcase <кейс> Устанавливает кейс на выбранный блок
/pncases delcase <кейс> Удаляет блоки кейса без удаления настроек
/pncases givekey <игрок> <ключ> <кол-во> Выдаёт игроку ключи
/pncases takekey <игрок> <ключ> <кол-во> Забирает ключи у игрока
Награды

Любой тип приза в одной системе

Настраивайте шанс, редкость, предмет для отображения, сообщение игроку и фактическое действие награды.

Предметная награда ITEM 
rewards:
  - chance: 45
    rarity: COMMON
    type: ITEM
    item:
      material: DIAMOND
      amount: 8
      name: '&bАлмазы &8x8'
Денежная награда VAULT 
rewards:
  - chance: 30
    rarity: RARE
    type: VAULT
    vault:
      amount: 2500
    visual:
      material: GOLD_INGOT
      name: '&e2500 монет'
    message: '&aВы получили &f{amount}&a!'
Тип Назначение Зависимость
ITEM Выдача предмета Не требуется
COMMAND Выполнение серверной команды Не требуется
VAULT Пополнение денежного баланса Vault и экономика
PLAYERPOINTS Выдача поинтов PlayerPoints
LUCKPERMS Выдача группы или прав LuckPerms
Анимации

Каждое открытие выглядит по-разному

Игрок выбирает анимацию через GUI, а выбор автоматически сохраняется в SQLite.

Наковальня

Падение наковальни и появление награды.

Динамит

Полёт TNT, взрыв и эффектная выдача приза.

Портал

Портальные эффекты и материализация награды.

Отравление

Ядовитый слайм, частицы и появление приза.

Астральный разлом

Магический разлом с вращением наград.

Настройка направления слайма 
animation:
  poison:
    slime-facing: PLAYER
    slime-pitch: 0

Допустимые значения: NORTH, SOUTH, EAST, WEST, PLAYER или собственное значение yaw.

Витрина кейса

Витрина отображает предмет над свободным кейсом. Предмет выбирается в /pncases machine, а эффекты можно отключать независимо от самой витрины.

Пример конфигурации 
idle-particles:
  enabled: true
  effects: true
  style: AURORA
  theme: MAGIC
  interval_ticks: 2
  radius: 0.85
  height: 1.35
  speed: 0.14
  view_distance: 28
  item:
    material: NETHER_STAR
    name: "&aДонат кейс"

Голограммы

pnCases автоматически выбирает установленный провайдер. Без плагина голограмм система кейсов продолжает работать.

  • FancyHolograms
  • DecentHolograms
  • Без обязательного TextDisplay fallback
Пример конфигурации 
hologram:
  enabled: true
  type: TEXT
  y: 1.5
  lines:
    - "&a&lДонат кейс"
    - "&7ПКМ, чтобы открыть"

Проверка обновлений

Система работает автоматически и не требует отдельной настройки в config.yml.

  • update-manifest.json
  • Последний GitHub Release
  • Git tags
  • Версия из plugin.yml

Файлы плагина

Конфигурация, сообщения и данные находятся в одной директории. Старые YAML-данные мигрируют в SQLite автоматически. 

plugins/pnCases/
├── config.yml
├── messages.yml
└── data.db
