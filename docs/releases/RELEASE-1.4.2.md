# pnCases 1.4.2

Hotfix после 1.4.1.

## Исправлено

- Добавлена команда `/pncases delcase <кейс>` для удаления привязки кейса к блоку.
- Добавлены алиасы команды: `/pncases deletecase`, `/pncases removecase`, `/pncases unsetcase`.
- Команда удаления убирает только `cases.<case>.block`, но сохраняет GUI, награды, стоимость и остальные настройки кейса.
- Исправлены старые анимации Наковальня, Динамит и Портал: теперь они показывают визуал именно выбранной награды, а не первый предмет из `animation.items`.
- Общая логика визуала наград вынесена для всех анимаций: сначала используется `visual`, затем совпадение из `animation.items`, затем безопасный fallback.
- Preview наград теперь может брать правильный предмет из `animation.items` для старых конфигов без `visual`.
- Из preview LuckPerms-наград убраны технические строки `group` и `node`.
- Если у LuckPerms-награды нет своего имени, но найден visual-предмет, в GUI, анимации, истории и сообщениях используется имя этого предмета. Если visual нет, показывается нормальное название `Привилегия`, а не техническая группа.
- История открытий теперь пытается показывать иконку самой награды вместо одинаковой книги.
- Кнопка preview и info-предмет preview получили отдельные иконки, чтобы не сливаться с историей.

## Дополнительный hotfix

- Исправлена загрузка кейсов после перезапуска сервера, если в `block.world` указан мир, которого сейчас нет на сервере.
- Такие кейсы больше не пропадают из памяти: настройки, награды, GUI и tab-complete остаются доступны.
- Отсутствующий мир теперь считается предупреждением, а не критической ошибкой config.yml.
- Голограммы безопасно пропускают кейсы без активного блока.
- Исправлен tab-complete для команд удаления: `delcase`, `deletecase`, `removecase`, `unsetcase`.
